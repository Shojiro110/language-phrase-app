# 要件定義書 — language-phrase-app（言語フレーズ学習アプリ）

> 書き方のヒントは各項目の「💡」に書いてあります。自分の言葉で空欄を埋めてください。
> 埋め終わったヒント行（💡）は消してOKです。

---

## 1. 背景・目的（なぜ作る）

海外出張が決まったため、外国の方と簡単な会話をするためによく使うフレーズと発音を覚える。文法などはフレーズを覚えた後にまなぶ

英語だけでなくドイツ語など他の言語を初めて学ぶ際にもある程度のフレーズを覚えておくと現地の人と打ち解けやすい


## 2. ターゲットユーザー（誰が使う）

まずは自分のみ



## 3. 解決したい課題・ユーザーストーリー（何に困っている）
日常でよく使うphraseを覚えたい
外国語の発音をまなびたい
すらすらと文章がでてくるようにしたい
新しい単語をまなびたい
-

## 4. 機能要件（何ができる）

### 必須（MVP）— まずここまで作る
💡 これが無いと成立しない機能。チェックボックス形式で。

- [ ]　フラッシュカード（日本語とそれを翻訳した語が表裏に表示）
- [ ]　正しい発音を学べる
- [ ]　正誤記録、間違ったものだけを復習

### あったら良い（2周目以降）

- [ ]　言語の切り替え（英語・ドイツ語など複数言語のフレーズを管理）
- [ ]　カテゴリ／タグでの絞り込み（あいさつ・食事・買い物 など）
- [ ]　学習進捗の可視化（覚えた数 / 全体数、達成率バー、学習した日付）
- [ ]　苦手カードの優先出題（間違えた回数が多い順に並べ替え）
- [ ]　例文・メモ欄（フレーズの使う場面を補足）
- [ ]　発音速度の調整・聞き取り（自分の発音を録音して聞き比べ）

## 5. 非機能要件（どんな品質で）

- 対応環境：スマホ（モバイルブラウザ）で快適に使えること。PC ブラウザでも利用可。
- 使いやすさ：シンプルなレイアウトで、片手・少ないタップで学習を回せること。
- ログイン：当面はログイン不要（自分専用のため）。将来、複数端末で同期したくなったら認証を検討。
- パフォーマンス：カード数が数百件程度でも快適に動くこと。
- データ：リロードやアプリ再起動でも学習データが消えないこと。

## 6. 画面一覧・画面遷移（どんな画面があるか）

- 画面①：ホーム／カード一覧 … 登録済みフレーズの一覧。学習開始ボタン、カード追加ボタン。
- 画面②：カード追加・編集 … 表（日本語）、裏（翻訳語）、言語、カテゴリを入力して保存。
- 画面③：学習（フラッシュカード） … 表を表示 → タップで裏返し → 発音再生 →「覚えた／まだ」を記録。
- 画面④：結果・進捗 … 今回の正誤、覚えた数 / 全体数、達成率バー、間違えたカードの復習へ。

遷移の流れ：

```
ホーム/一覧 ──[学習開始]──> 学習(フラッシュカード) ──[最後まで]──> 結果・進捗
   │  ↑                                                          │
   │  └──────────────[間違いだけ復習]────────────────────────────┘
   │
   └──[追加/編集]──> カード追加・編集 ──[保存]──> ホーム/一覧
```



## 7. データ設計（何を保存するか）

**cards テーブル**（フレーズ1件＝1行）

| 項目名 | 型 | 説明 |
|--------|----|----|
| id | uuid | カードの一意なID（自動採番） |
| front | text | 表：日本語の語・フレーズ |
| back | text | 裏：翻訳した語・フレーズ |
| language | text | 学習言語（例: en, de） |
| category | text | カテゴリ／タグ（例: あいさつ）※任意 |
| learned | boolean | 覚えたかどうか（true/false） |
| wrong_count | integer | 間違えた回数（復習の優先度に使用） |
| created_at | timestamp | 作成日時 |
| last_reviewed_at | timestamp | 最後に学習した日時 ※任意 |

> 学習の正誤や日付は当面 cards に持たせる。履歴を細かく残したくなったら別途
> `study_logs`（card_id・正誤・日時）テーブルを追加する。

## 8. 技術スタック（何で作る）

| 役割 | 技術 |
|------|------|
| フレームワーク | Next.js（App Router） |
| 言語 | JavaScript |
| データ保存 | Supabase（予定。MVP段階はブラウザの localStorage でも可） |
| バージョン管理 | Git / GitHub |
| 公開 | Vercel（予定） |
| 発音 | Web Speech API（ブラウザ標準の読み上げ） |

## 9. スコープ外（あえて作らないこと）

- ユーザー登録・ログイン認証（自分専用のため当面なし）
- 他ユーザーとの共有・SNS的な機能
- 文法解説や学習コンテンツの自動生成（フレーズ暗記に集中）
- AIによる翻訳・自動カード生成（手動で登録する）
- ネイティブアプリ化（まずは Web のみ）

## 10. 完成の定義（どうなったら"完成"か）

スマホのブラウザで、フレーズをカード登録 → フラッシュカードで表裏を学習 →
発音を再生 →「覚えた／まだ」を記録でき、リロードしてもデータが残り、
間違えたカードだけを復習できる状態になったら MVP 完成とする。
